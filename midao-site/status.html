Status / future

Current status of the project and it's improvement plan

  • Current Status
  • Future

1. Current status

Current version of Midao JDBC is 0.9.4

Midao JDBC is 0.9.5 is planned to be released in August-September and will come with xml input/output handler

Change log can be viewed from: GitHub

2. Future

Midao JDBC

1. (done) Java 1.5 support

2. (done) Functional tests with older JDBC drivers

3. (done) C3P0

4. (done) Remove any strict dependency

Possible 0.9 release

5. (done) Microsoft SQL support

Possible 0.9.1 release

6. (done) Spring Exception Handler

7. (done) Lazy loading for UniversalTypeHandler and BaseStatementHandler. This would allow to supply huge input(for TypeHandler) and return huge output(for BaseStatementHandler) with small memory usage. In case of huge output - manual transaction handling would be required.

Possible 0.9.2 release

8. (done) Create new scrollable and updateable Lazy Output handlers

Possible 0.9.3 release

11. (WIP) Midao*(MjdbcFactory, MjdbcConfig etc.) classes deprecated to avoid ambiguity. Please use Mjdbc*(MjdbcFactory, MjdbcConfig) from now on. Midao*Exception was renamed into Mjdbc*Exception.

Possible 0.9.4 release

9. (WIP) Create new Input handlers which would allow to use named queries from external files (planned support of iBatis/myBatis xml mappings)

10. (WIP) Create new Output handlers which would allow to use named queries from external files (planned support of iBatis/myBatis xml mappings)

Possible 0.9.5 release. ETA August

12. Improve output handlers and allow mapping bean fields similar to JPA: @Column(name="SQL_COLUMN_NAME", <other>)

Possible 0.9.6 release.

13. Final code check, cleanup and consistency check. JavaDoc improvements and code comments removal.

14. XML and Array functional tests for all databases (where applicable)

Possible 0.9.9 release. ETA September
Possible 1.0.0 release. ETA October

15. More (independent) examples

16. SQL Struct support (TypeHandler)

17. More and more functional tests

18. More unit tests

19. Focus more on ORM capabilities. Goal is not to make another ORM library, but to make it more comfortable to use object relational mapping when needed

J. TBD

To be announced. ETA: Autumn-Winter 2013
To be announced. ETA: Winter-Spring 2014
